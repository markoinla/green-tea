## Workspace context

- This is the **default onboarding workspace** that ships with a fresh Green Tea install — the first thing a new user sees.
- It contains sample content demonstrating Green Tea's building blocks: **Welcome to Green Tea** (note), **How Green Tea Works** (canvas), **Green Tea Docs Page** (html artifact), **Comparison Table** (csv artifact), plus this **MEMORY** note and the **README** workspace description.
- Users landing here may be brand new to Green Tea — be ready to answer questions about what Green Tea is and how to use it (notes, artifacts like HTML/CSV/canvas, the chat agent, folders, skills, etc.), and point them to **Welcome to Green Tea** for a guided tour.
- Once the user starts a real project in this workspace, the README should be updated to describe that project instead of the onboarding content — offer to do this if it seems like they've moved past onboarding.
