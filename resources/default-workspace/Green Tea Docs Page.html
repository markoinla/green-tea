Welcome to Green Tea

Green Tea is the open source AI workspace for people who want AI that actually does things without giving up control of their data.

Working with HTML documents

This page is an HTML artifact: a polished, rendered document that lives directly in your workspace as a plain .html file. HTML artifacts are useful for reports, dashboards, briefings, slide-style documents, onboarding guides, and anything that benefits from layout, navigation, tables, or light interactivity.

Rendered in Green TeaOpen HTML files in the workspace and they display as finished documents, not raw code.
Editable with the agentAsk the agent to revise the copy, layout, sections, styling, or data and it can update the same file.
ShareableUse Share to create a web version you can send to other people.

Use the HTML toolbar

When an HTML document is open, the toolbar gives you a few document-specific controls:

ButtonWhat it doesWhen to use it
ShareCreates a shareable web version of the HTML document, including an editable version when you want someone else to collaborate or make changes.Use this when you want to send a polished page outside your workspace.
EditTurns on direct text editing inside the rendered web page.Use this for quick copy edits without going through the raw HTML.
SelectLets you click an element on the page and send that exact selection to the chat.Use this when you want the agent to know precisely which heading, paragraph, card, table, or section to change.
ReloadRefreshes the rendered page from the current file.Use this after changes if the page does not update automatically.

Editing HTML documents

HTML artifacts are rendered documents. For quick changes, click Edit and update text directly on the page. For larger changes, ask the agent: “add a section,” “make this more concise,” “update the table,” or “restyle this as a briefing.” If you need the agent to work on a specific part of the page, click Select, choose the element, and send it to chat so the agent has exact context.

Sharing HTML documents

When you are ready to share, click Share to create a web version of the artifact. Shared HTML documents can include an editable version for collaborators, while still keeping your local workspace file as the source of truth.

Getting started

  1. Choose an AI model — use Green Tea's built-in model, or log in with your Anthropic or OpenAI account.
  2. Write a note — use this workspace like a normal outliner for ideas, drafts, meeting notes, specs, or research.
  3. Ask the agent for help — open the chat sidebar and mention a note with @ when you want the agent to use it as context.
  4. Create an artifact — ask Green Tea to make a table, HTML report, dashboard, canvas diagram, checklist, or slide-style page.
  5. Try automation — ask the agent to run a recurring task, like a weekly research digest or workspace summary.
  6. Make it yours — enable skills, connect providers, customize the theme, and organize around your own projects.

The big idea

Most tools make you choose between a notes app that owns your knowledge, an AI chat tool that forgets your context, or a SaaS workspace that locks your workflows into someone else's cloud.

Use Green Tea as a writing and thinking space, but also as a place where agents can research, organize, generate, update, and publish work for you.

Understand your workspace

Your workspace is a folder

A Green Tea workspace is just a folder on disk. Notes are Markdown. Tables are CSV. Artifacts are ordinary files. You can open them in Green Tea, Finder, Git, Obsidian, VS Code, or any other tool that understands plain files.

  • Local-first by default — your workspace lives on your machine.
  • Portable formats — Markdown notes, CSV tables, images, PDFs, HTML, and other file-based artifacts.
  • No lock-in — if Green Tea disappeared tomorrow, your files would still be yours.
  • Version history — workspaces are automatically version-controlled with Git, so you can restore earlier versions.

Organizing your work

  • Workspaces are top-level folders on disk.
  • Folders organize notes and artifacts inside a workspace.
  • README.md stores stable workspace context.
  • MEMORY.md stores the agent's running memory.

Finding things

  • Cmd/Ctrl-K — command palette: jump to any note across every workspace.
  • Cmd/Ctrl-F — find and replace within the current note.

Work with notes and artifacts

Writing notes

Notes are outlines. Every line is a block you can nest, fold, and rearrange.

  • Press Enter for a new block, Tab to indent, Shift-Tab to outdent.
  • Move a block with Cmd/Ctrl-Shift-↑ and Cmd/Ctrl-Shift-↓.
  • Type / at the start of a block to insert text, headings, task lists, blockquotes, code blocks, tables, and images.
  • Use inline formatting like bold, italic, inline code, highlight, and links.

Task lists are just blocks with checkboxes:

  • Read this welcome note
  • Ask the agent to draft or reorganize something
  • Create your first artifact
  • Make this workspace your own

Linking ideas

Type [[ to link to another note by title. Links show up in reverse on the target note as backlinks. Links are stored as plain [[Title]] text on disk, so they stay portable and Obsidian-compatible.

Use the AI agent

How to work with the agent

Open the chat sidebar and talk to the agent. It can work across your workspace, not just the current note.

  • Mention notes with @ to pull them into context.
  • Attach files or images when you want the agent to inspect or transform them.
  • The agent can read and search notes, query by tag/property, create notes and folders, generate artifacts, and reorganize the workspace.
  • It can search the web, read pages, write code, run tools, and create new capabilities for itself.
  • In Review mode, edits are proposed as a diff. In Auto mode, approved operations can happen immediately.

What the agent can do

  • Summarize, rewrite, or restructure a messy note.
  • Research a topic and draft a briefing.
  • Turn a braindump into an organized outline, table, or checklist.
  • Generate an HTML report, dashboard, slide deck, or canvas diagram.
  • Create a recurring workflow, like a weekly research digest.
  • Build a custom skill or plugin for your workflow.

Automate and extend Green Tea

Scheduled agents

Green Tea can run agent tasks in the background. Ask it to do something recurring, like:

Every Monday at 9am, summarize what changed in this workspace into a new note.

Scheduled agents make Green Tea a workspace that can monitor, update, research, and prepare work while you are away.

Memory that carries over

The agent keeps lightweight memory in MEMORY.md — facts, preferences, decisions, and project context it should remember across conversations.

Skills

Skills are reusable capability packs the agent loads on demand. Green Tea ships with skills for PDF, DOCX, XLSX, and PPTX files, plus diagrams, slide decks, intelligence briefings, kanban boards, and more.

  • skill-creator — build a brand-new agent skill.
  • plugin-creator — build a new artifact viewer/editor.

Plugins

Plugins add new artifact types with their own viewers. Bundled plugins include kanban boards, Mermaid diagrams, and todo lists. Plugins are sandboxed to their own files.

Control, sharing, and customization

Open source, provider login, and self-hosting

  • Open source — inspect the code, contribute, fork it, or self-host the pieces you need.
  • Use your existing AI plans — log in with Anthropic or OpenAI and use your plan inside Green Tea.
  • Local-first data — your notes and artifacts live on your machine by default.
  • Optional cloud convenience — use hosted services when you want sync, managed AI, or publishing convenience.

Sharing and publishing

Publish any note or shareable artifact to the web as a read-only snapshot. You get a clean link at share.greentea.app. Shares expire after 30 days, and re-publishing resets the clock.

Make it yours

Open Settings to choose your AI provider/model, log in with Anthropic or OpenAI, connect other providers, toggle Review vs. Auto approval, enable skills/plugins/MCP servers, and customize the theme.
